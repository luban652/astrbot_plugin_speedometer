# astrbot_plugin_speedometer

一款专为 AstrBot 设计的**测速与周回计算**插件。通过简单的指令，用户可以精准记录操作间隔，自动换算为“每小时周回数”，非常适合需要记录游戏副本效率、工作节奏或重复性任务耗时的场景。

## 插件简介

**astrbot_plugin_speedometer** 能够记录用户发送指令的时间戳，并计算相邻两次记录之间的时间差。它不仅以“分:秒”的格式直观展示耗时，还能根据当前效率预测一小时内可完成的次数（周回数）。插件支持多轮连续记录，并能在结束时提供完整的统计报告。

## 功能说明

* **实时耗时计算**：自动计算本次 `/cs` 与上次 `/cs` 的精确时间差。
* **周回效率换算**：根据当前时间间隔，计算出 1 小时内理论上可以完成的循环次数。
* **多轮记录支持**：支持不间断地多次发送指令，插件将始终以“后一次”对比“前一次”进行增量计算。
* **统计总结报告**：使用 `/cse` 停止记录后，插件会汇总本次所有记录的平均耗时、最快/最慢记录及总次数。

## 插件流程

插件内部维护一个会话级的时间状态机，其核心逻辑如下：

1.  **触发阶段**：
    * 当接收到 `/cs` 指令时，插件首先检查当前会话（`session_id`）是否已有活跃的计时器。
2.  **逻辑处理阶段**：
    * **首次发送**：若为首次（或 `/cse` 后的第一次），插件记录当前系统时间戳为 `start_time`，并初始化记录列表，回复“计时开始”。
    * **后续发送**：计算 `当前时间` - `上次记录时间`。
        * 将差值转换为 `M分S秒` 格式。
        * 计算周回数：$3600 / 间隔秒数$（取整）。
        * 更新“上次记录时间”为当前时间，并将此段耗时存入历史记录。
3.  **终止阶段**：
    * 当接收到 `/cse` 指令时，插件提取历史记录列表。
    * 计算总耗时、平均耗时、最大/最小耗时。
    * 清空当前会话的计时状态，并输出汇总 Markdown/纯文本报告。

## 使用方法

### 1. 开始与连续记录
发送 `/cs` 即可触发记录。

* **第一次发送**：
    > 🤖：测速开始！已记录起始时间点。
* **第二次及以后发送**：
    > 🤖：第 2 次记录成功！
    > ⏱️ 间隔耗时：0分45秒
    > 🔄 当前周回：80.0 次/小时
    > (基准：上次记录时间 14:30:00)

### 2. 结束并查看总结
当你完成所有轮次的测速后，发送 `/cse`。

* **发送 `/cse`**：
    > 🤖：测速已停止。本次统计报告：
    > 📊 总计次数：5 次
    > ⏳ 平均耗时：1分10秒
    > ⚡ 最快纪录：0分55秒
    > 🐢 最慢纪录：1分30秒
    > 🏁 状态：已重置，发送 /cs 可重新开始。

## 配置说明

本插件目前为开箱即用版，无需额外配置 `_conf_schema.json`。所有计时数据均在内存中按会话隔离，确保多用户使用时互不干扰。

## 注意事项

* **会话隔离**：插件基于 `session_id` 进行识别。在群聊中，所有成员的操作可能共用一个计时状态（取决于 AstrBot 的 unique_session 配置），建议在私聊或特定环境下使用以获得准确数据。
* **超时机制**：若长时间未发送下一次 `/cs`，计时器将持续挂载。建议在任务结束后务必发送 `/cse` 释放资源。
* **精度限制**：周回数计算基于秒级精度进行四舍五入。
* **合规性**：本插件严禁用于任何形式的非法自动化脚本监控或违反平台规则的行为。
